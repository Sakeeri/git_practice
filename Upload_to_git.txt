- 작업 디렉토리 : 현재 내 컴퓨터에서 작업하는 실제 파일들
- 인덱스 : 준비영역
- 헤드 : 최종 확정본(=commit)


1. 변화가 생겼다.
2. git status로 확인 - 빨간색이면 변화 있음
3. git add 파일명 or * 로 add : 인덱스에 추가
4. git commit '커밋 메세지' : 헤드에 추가. 변경 내용 저장(세이브 포인트)
5. git push [저장소이름] [브랜치이름]: 원격 저장소(깃허브)에 저장

1. git pull : 원격 저장소에 있는 내용과 로컬 저장소에 있는 내용을 동기화시킨다(다운받는다)
2. git fetch : 차이점을 확인만 하고 병합은 하지 않는다.
pull = fetch + merge

화면 초기화 : Ctrl + L
한 행의 처음과 끝 : Ctrl + A, Ctrl + E
목록 보기 : ls 또는 dir
파일의 내용 보기 : cat
특정 문자를 검색 : grep
디렉터리로 이동 : cd
디렉터리 생성 : mkdir
파일 삭제 : rm
파일 생성 : touch
